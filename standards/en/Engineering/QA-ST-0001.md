# QA-ST-0001 — Test and Quick Verification Standard

This standard is maintained by the QA department.

**[→ View Complete Standard](../QA/QA-ST-0001.md)**

---

## Quick Summary

QA-ST-0001 defines how to execute quick, consistent, and reproducible tests, ensuring that any internal or external delivery maintains the minimum quality level expected by DJIN Tech.

It covers:
1. Environment and Credentials
2. Translations (DeepL + Takoboto)
3. Spellcheck (Code + UI)
4. Basic Data Verification
5. Interface Translation
6. Leaks (UUIDs, stacktraces, undefined)
7. Layout (Desktop + iPhone 390px)
8. Main Flow (Create → Edit → Delete)
9. Spelling and Messages
10. Colors and Branding
11. Evidence (Screenshots + Videos)

**Executed by:** DJIN Members (QA) and DJIN Members (Engineering) during development

**Operation Modes:** Strict (Complete) / Lazy (Quick)

**Outputs:** Documentation + Handoff OR Immediate Fix + Log
