# QA-ST-0002 — Standard de Qualidade para DJIN Members (QA) — Versão Operacional

Este documento define **os requisitos mínimos** para que qualquer solicitação do cliente possa avançar para Arquitetura.

Ele é aplicado por **DJIN Members (QA)** em **QA[1] (entrada)** e **QA[2] (validação final)**.

A ausência de qualquer requisito listado implica **bloqueio**, retorno ao cliente e solicitação de esclarecimentos.

---

## 👤 Quem Executa Este Standard

Este standard é aplicado por **DJIN Members** responsáveis pela triagem de qualidade (QA).

Qualquer DJIN Member em função de QA deve seguir estas diretrizes ao avaliar solicitações de clientes.

---

# 1. FEATURE / AJUSTE / REMOÇÃO DE FEATURE

Para ser considerado válido, o pedido deve conter:

1. **Propósito claro** – qual objetivo o cliente quer atingir.
2. **Problema sendo resolvido** – qual dor ou falha a mudança resolve.
3. **Localização no sistema** – tela, funcionalidade ou fluxo afetado.
4. **Imagens ilustrativas** (quando fizer sentido).
5. **Fluxograma do fluxo desejado** (quando fizer sentido).
6. **Quem irá utilizar** e **exemplo de uso real**.

### Mensagem padrão ao cliente quando faltar informações

```
ご連絡ありがとうございます。
大変恐れ入りますが、本件の目的や解決したい課題が明確に確認できなかったため、追加のご説明をお願いできますでしょうか。
また、どの画面・どの機能に対するご依頼か、併せて画像や具体例などをご共有いただけますと大変助かります。
お手数をおかけいたしますが、よろしくお願い申し上げます。
```

---

# 2. PEDIDO DE DADOS

Requisitos mínimos:

1. **Shape dos dados** (colunas, formato, exemplos se possível).
2. **Ambiente** (青 / 緑).
3. **Filtros** – intervalo de tempo, condições, limites, etc.

### Mensagem padrão

```
ご依頼いただきありがとうございます。
必要な情報が一部不足しているため、以下についてご教示いただけますでしょうか。
・必要なデータ項目（カラム）
・対象の環境（青／緑）
・範囲および適応条件など
お忙しいところ恐縮ですが、ご確認のほどよろしくお願いいたします。
```

---

# 3. BUG REPORT

Requisitos mínimos:

1. **Onde ocorreu** (tela, URL, sistema).
2. **Quando ocorreu** (data e hora).
3. **Quem** (nome, e-mail, ID).
4. **O que era esperado**.
5. **O que aconteceu** (detalhado; imagens/vídeo ajudam e evitam retrabalho).

### Mensagem padrão

```
ご連絡ありがとうございます。
状況を正確に把握するため、以下の情報をご教示いただけますでしょうか。
・発生した画面またはURL
・発生日時
・対象者のお名前／ID/メールアドレス
・期待されていた動作
・実際に発生した事象（可能でしたら画像・動画もご共有ください）
いただいた情報をもとに速やかに調査を進めさせていただきます。
何卒よろしくお願い申し上げます。
```

---

# 4. PERGUNTAS DO CLIENTE

Requisitos mínimos:

1. **Contexto claro** – a pergunta tem fundamento ou referência?
2. **Task ID** (se existir).
3. **Localização** – qual tela/fluxo a pergunta envolve.
4. **Coerência** – a pergunta faz sentido no contexto apresentado?

### Mensagem padrão

```
ご質問ありがとうございます。
恐れ入りますが、本件の文脈が確認できなかったため、
以下についてご教示いただけますでしょうか。
・本質問に関連するタスクID
・どの画面／どの処理についてのご質問か
・背景または経緯
お手数をおかけしますが、正確に確認するため、
ご協力をお願いいたします。
```

---

# 5. PEDIDO DE DEPLOY

Requisitos mínimos:

1. **Segunda ou terça-feira** como data preferencial.
2. **Confirmação de testes no 緑 (customer-preview)**.
3. **Arquitetura define** se o deploy ocorre de dia ou de noite.

### Mensagem padrão

```
ご依頼ありがとうございます。
青い環境への反映に際し、以下の点を確認させてください。
・反映日は「月曜」または「火曜」で問題ございませんか
・緑環境での動作確認は完了しておりますでしょうか。
確認の上、スケジュールを確定させていただきます。
引き続きよろしくお願い申し上げます。
```

---

# 6. PEDIDO DE ALINHAMENTO

Casos possíveis:

### (i) Prazo ausente

```
ご連絡ありがとうございます。
念のため確認させていただきたいのですが、
本件のご希望の納期（締切日）はいつ頃を想定されておりますでしょうか。
正確なスケジュール調整のため、ご教示いただけますと幸いです。
```

### (ii) Prioridade ausente

```
ご依頼ありがとうございます。
本件の優先度につきまして、【優先順位1～3】、または「緊急」など、
ご希望のレベルをご教示いただけますでしょうか。
優先度に応じて社内の対応順序を調整させていただきます。
```

### (iii) Confirmar entendimento

```
ご連絡ありがとうございます。
正確に対応するため、本件の意図およびご要望内容が
合っているか確認させてください。
（※こちらで理解している内容を簡単に箇条書き）
上記に相違がないかご確認いただけますでしょうか。
問題なければ、開発部に共有して進めさせていただきます。
```

### (iv) Mudança de direção

```
ご連絡ありがとうございます。
念のため確認ですが、
「現在進めている作業を一旦停止し、別の作業を優先する」
というご指示でお間違いございませんでしょうか。
本件はプロジェクト全体に影響するため、ご教示いただけますと幸いです。
ご確認のほどよろしくお願いいたします。
```

---

# 7. RESPOSTA DO CLIENTE (INCOMPLETA)

Critério:

* Respondeu apenas parte do que foi solicitado.

### Mensagem padrão

```
ご返信ありがとうございます。
いただいた内容のうち、以下の点についてまだ確認できておりませんため、
追加のご回答をお願いできますでしょうか。
・（※未回答の項目を箇条書き）
お手数をおかけいたしますが、よろしくお願いいたします。
```

---

# 8. RUÍDO

### (i) Mensagem impossível de entender

```
ご連絡ありがとうございます。
恐れ入りますが、いただいた内容の意図を正確に理解できませんでした。
お手数ですが、もう少し詳しくご説明いただけますでしょうか。
正確に対応するため、ご協力をお願いいたします。
よろしくお願い申し上げます。
```

### (ii) Muitos temas misturados

```
ご連絡ありがとうございます。
本件、複数のテーマが一度に含まれているように見受けられましたため、
正確に確認するためにも、以下のように話題を分けていただけますでしょうか。
（例：①◯◯ ②△△ ③□□）
お手数をおかけいたしますが、ご協力いただけますと幸いです。
よろしくお願い申し上げます。
```

---

# 9. NOVA CATEGORIA

Se algo não se encaixar em nenhuma categoria existente, consultar o departamento de Arquitetura/Diniz.

---

# 10. REGRA SUPREMA — "NA DÚVIDA, PERGUNTE"

Se houver qualquer dúvida sobre categoria, requisitos mínimos, intenção do cliente ou coerência da solicitação:

* **Não avance para Arquitetura**
* **Não assuma nada**
* **Solicite esclarecimentos**

Mensagem universal:

```
ご連絡ありがとうございます。
念のため確認させていただきたいのですが、
本件についての意図や背景が明確に読み取れなかったため、
もう少し詳しくご説明いただけますでしょうか。
正確に対応するため、ご協力をお願いいたします。
```
