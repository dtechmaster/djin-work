## 🧩 1. Julgamento da organização da informação recebida (shape check)

O QA deve avaliar se a informação recebida está organizada o suficiente para ser processada.

Independente da categoria, é obrigatório verificar:

- ✅ Clareza geral
- ✅ Estrutura mínima
- ✅ Propósito explícito
- ✅ Localização no sistema
- ✅ Forma como o cliente apresentou o pedido
- ✅ Presença ou ausência de anexos/imagens
- ✅ Contexto suficiente para evitar ruído

---

## 🛠 2. Requisito universal para qualquer solicitação de feature/ajuste

Se o cliente solicitou uma nova feature ou ajuste, deve haver:

- ✅ **Explicações claras** do que deseja
- ✅ **Propósito** do pedido
- ✅ **Qual problema está sendo resolvido?**
- ✅ **Onde no sistema** a alteração será feita
- ✅ **Imagens ilustrativas**
- ✅ **Fluxograma** (mesmo simples) mostrando a ordem/fluxo desejado
- ✅ **Quem irá utilizar**
- ✅ **Um exemplo concreto de uso**

### 📩 Mensagem padrão se não atender aos requisitos mínimos

```
ご連絡ありがとうございます。
大変恐れ入りますが、本件の目的や解決したい課題が明確に確認できなかったため、
追加のご説明をお願いできますでしょうか。
また、どの画面・どの機能に対するご依頼か、併せて画像や具体例などをご共有いただけますと大変助かります。
お手数をおかけいたしますが、よろしくお願い申し上げます。
```

> ⚠️ **Observação**: Pedidos de **remoção de feature** seguem exatamente os mesmos requisitos.

## 📊 3. Pedido de Dados

O cliente deve informar:

- ✅ **Shape dos dados** (colunas, formato, etc.)
- ✅ **Ambiente** (azul/verde)
- ✅ **Filtros** (extensão, período, condições, limites)

### 📩 Mensagem padrão se faltar informação

```
ご依頼いただきありがとうございます。
必要な情報が一部不足しているため、以下についてご教示いただけますでしょうか。
・必要なデータ項目（カラム）
・対象の環境（青／緑）
・範囲および適応条件など
お忙しいところ恐縮ですが、ご確認のほどよろしくお願いいたします。
```

## 🐞 4. Caso seja reportado um Bug

Obrigatório informar:

- ✅ **Onde ocorreu** (tela, URL, etc.)
- ✅ **Quando ocorreu** (data/hora)
- ✅ **Quem** (nome, e-mail, ID)
- ✅ **O esperado**
- ✅ **O que aconteceu** (descrição clara; imagens/vídeos opcionais, mas desejáveis)

### 📩 Mensagem padrão se faltar informação

```
ご連絡ありがとうございます。
状況を正確に把握するため、以下の情報をご教示いただけますでしょうか。
・発生した画面またはURL
・発生日時
・対象者のお名前／ID/メールアドレス
・期待されていた動作
・実際に発生した事象（可能でしたら画像・動画もご共有ください）
いただいた情報をもとに速やかに調査を進めさせていただきます。
何卒よろしくお願い申し上げます。
```

## ❓ 5. Se o cliente fez uma pergunta

QA deve verificar:

- ✅ **Existe contexto claro da pergunta?**
  - Ex.: veio no LINE solto, veio como comentário dentro de uma task
- ✅ **Há Task ID relacionado?**
- ✅ **A pergunta se encaixa no contexto apresentado?**
  - Ou está deslocada e usa termos inexistentes?

### 📩 Mensagem padrão se faltar informação

```
ご質問ありがとうございます。
恐れ入りますが、本件の文脈が確認できなかったため、
以下についてご教示いただけますでしょうか。
・本質問に関連するタスクID
・どの画面／どの処理についてのご質問か
・背景または経緯
お手数をおかけしますが、正確に確認するため、
ご協力のほどよろしくお願いいたします。
```

---

## 🚀 6. Pedido de Deploy para Produção

O QA deve confirmar:

- ✅ **Agendar para segunda ou terça-feira**
- ✅ **Confirmar com Arquitetura** se o deploy é diurno ou noturno
- ✅ **Passou pelo customer-preview?**
- ✅ **Cliente testou?**
- ✅ **Era obrigatório que tivesse testado?**

### 📩 Mensagem padrão se faltar informação

```
ご依頼ありがとうございます。
青い環境への反映に際し、以下の点を確認させてください。
・反映日は「月曜」または「火曜」で問題ございませんか
・緑環境での動作確認は完了しておりますでしょうか。
確認の上、スケジュールを確定させていただきます。
引き続きよろしくお願い申し上げます。
```

## 🔧 7. Se o cliente solicitou um alinhamento

QA deve garantir:

### 7.1 — Prazo está claro

Se não houver:

```
ご連絡ありがとうございます。
念のため確認させていただきたいのですが、
本件のご希望の納期（締切日）はいつ頃を想定されておりますでしょうか。
正確なスケジュール調整のため、ご教示いただけますと幸いです。
```

### 7.2 — Prioridade está definida

Se não houver:

```
ご依頼ありがとうございます。
本件の優先度につきまして、【優先順位1～3】、または「緊急」など、
ご希望のレベルをご教示いただけますでしょうか。
優先度に応じて社内の対応順序を調整させていただきます。
```

### 7.3 — Entendimento está alinhado

Se houver dúvida:

```
ご連絡ありがとうございます。
正確に対応するため、本件の意図およびご要望内容が
合っているか確認させてください。
（※こちらで理解している内容を簡単に箇条書き）
上記に相違がないかご確認いただけますでしょうか。
問題なければ、開発部に共有して進めさせていただきます。
```

### 7.4 — Mudança de direção

Confirmar explicitamente:

```
ご連絡ありがとうございます。
念のため確認ですが、
「現在進めている作業を一旦停止し、別の作業を優先する」
というご指示でお間違いございませんでしょうか。
本件はプロジェクト全体に影響するため、ご教示いただけますと幸いです。
ご確認のほどよろしくお願い申し上げます。
```

### 7.5 — Conexão com Task

⚠️ **Tudo deve ficar dentro da Task quando existir uma.**

## 📬 8. Se o cliente enviou uma resposta / suposta resposta

QA deve verificar:

- ✅ **Está claro O QUE está sendo respondido?**
- ✅ **A resposta está completa?** (3 itens enviados → 3 respondidos)
- ✅ **Há link, menção ou reply direto?**

### 📩 Mensagem padrão se incompleta

```
ご返信ありがとうございます。
いただいた内容のうち、以下の点についてまだ確認できておりませんため、
追加のご回答をお願いできますでしょうか。
・（※未回答の項目を箇条書き）
お手数をおかけいたしますが、よろしくお願いいたします。
```

---

## 🧹 9. Se não se encaixa em nenhuma categoria → Ruído (Fallback)

### 9.1 — Mensagem impossível de entender

```
ご連絡ありがとうございます。
恐れ入りますが、いただいた内容の意図を正確に理解できませんでした。
お手数ですが、もう少し詳しくご説明いただけますでしょうか。
正確に対応するため、ご協力をお願いいたします。
よろしくお願い申し上げます。
```

### 9.2 — Mensagem com múltiplos temas (tumulto)

```
ご連絡ありがとうございます。
本件、複数のテーマが一度に含まれているように見受けられましたため、
正確に確認するためにも、以下のように話題を分けていただけますでしょうか。
（例：①◯◯ ②△△ ③□□）
お手数をおかけいたしますが、ご協力いただけますと幸いです。
よろしくお願い申し上げます。
```
